# AGENTS.md

## 프로젝트 설명

이 저장소는 CSV 파일을 데이터 저장소로 사용하는 Python CLI 가계부 앱 프로젝트입니다. 수입과 지출을 기록, 조회, 집계하고 명령줄에서 예산 흐름을 관리하는 것을 목표로 합니다.

## 코딩 규칙

- 모든 Python 함수, 메서드, 공개 API에는 타입 힌트를 반드시 작성합니다.
- 함수 하나는 최대 50줄을 넘기지 않습니다.
- 작고 명확한 함수로 분리하고, 입출력과 비즈니스 로직을 가능한 한 분리합니다.
- CSV 읽기/쓰기 로직은 예외 상황을 명시적으로 처리합니다.

## TDD 규칙

- 구현 전 반드시 실패하는 테스트를 먼저 작성합니다.
- 테스트가 실패하는 것을 확인한 뒤 최소 구현으로 통과시킵니다.
- 리팩터링은 테스트가 통과한 상태에서만 진행합니다.

## 품질 규칙

- 순환복잡도는 10 이하로 유지합니다.
- 조건문이 복잡해지면 함수 분리, 가드 절, 데이터 구조 개선을 우선 검토합니다.
- 중복 로직은 테스트를 보존한 상태에서 공통 함수로 추출합니다.

## 품질 검수 규칙

- 커밋 전 반드시 `qa_engineer` 서브에이전트로 품질 검수를 진행합니다.
- `qa_engineer`는 테스트, 복잡도, 타입 힌트, 함수 길이, CSV 오류 처리, CLI 동작 회귀를 확인해야 합니다.
- 품질 검수에서 발견된 문제는 커밋 전에 수정합니다.

## 테스트 및 품질 명령어

- 테스트 실행: `pytest`
- 순환복잡도 검사: `radon cc .`

## 커밋 규칙

- 하나의 기능이 개발되면 해당 기능 단위로 커밋합니다.
- 커밋 전 테스트와 품질 검수를 완료합니다.
- 커밋 후 원격 저장소로 푸시를 진행합니다.
